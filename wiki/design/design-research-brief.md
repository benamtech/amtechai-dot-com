# Design Research Brief

## Strategic translation

AMTECH should evolve from an AI agency website into a modern technical publishing house for small-business transformation. The visual system should combine field-manual utility, corporate standards-manual authority, editorial wit, and AI operating-system precision.

## Reference lineage

### Paul Rand

Use reduction, symbolic clarity, and memorable punctuation. The AMTECH red period should operate like a brand signal, not an ornament.

### AT&T / Bell corporate modernism

Use standards-driven confidence: consistent grids, canonical labels, repeated rules, and restrained visual identity. Pages should feel designed by an organization that can operate systems at scale.

### Otl Aicher / Ulm

Use pictograms as functional language. Icons should be readable as signage: simple geometry, single-color rendering, strong silhouettes, and clear semantic categories.

### Ladislav Sutnar

Structure business information so people can act quickly. Articles should behave like catalog entries for operating work: title, industry, task, inputs, steps, outputs, risks, next action.

### NASA 1970s standards

Use procedural confidence: document numbers, mission labels, diagrams, operating notes, status panels, and clear hierarchy. This is especially relevant for AI workflow pages that must make complex systems feel governable.

### Swiss grid

The grid is not background decoration. It is the machine that organizes authority. Use asymmetry, exact columns, hard section transitions, and clear alignment.

### Bauhaus / Bayer

Use typography as structure. Do not overdecorate. Let proportion, rule weight, scale, and contrast carry the page.

### Buckminster Fuller

Frame AI adoption as design science and future duty. The tone should imply that serious operators learn the new system because the operating environment has changed.

## Translation into web components

- `EditorialKicker`: mono uppercase orientation label.
- `EditorialHeaderRule`: AMTECH mark plus section label on a hard horizontal rule.
- `ManifestoHeadline`: large compressed statement with red punctuation.
- `FieldGuideCard`: dense, square-edged information unit.
- `ProofPanel`: oversized number, evidence copy, caption/source line.
- `OperatorChecklist`: actionable checklist with human/machine roles.
- `ScarletCTA`: red conversion block with clear assignment.
- `IndustrialQuote`: quote or doctrine block with red vertical rule.
- `ArticleIndexGrid`: catalog-style index for industry/task/skill pages.
- `PictogramStep`: procedure step with a signage-grade symbol.
