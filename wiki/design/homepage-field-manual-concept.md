# AMTECH Homepage — Field Manual Concept

> From-scratch homepage structure and copy for `amtechai.com`. This is intentionally not a rewrite of the current page. It is a standalone homepage concept built from the new AMTECH design system.

## Page identity

- Page type: Public homepage / operating-library gateway
- Document label: `AMTECH FIELD MANUAL / PUBLIC EDITION`
- Primary audience: Ambitious SMB owners and operators who know AI matters but do not know where it fits into daily work.
- Primary conversion: Book an operator briefing.
- Secondary conversion: Browse field guides.

## Section 02 — Doctrine plate

### Mono label

`OPERATING PREMISE 01`

### Headline

The next employee in your business may not be a person. But the operator is still in charge.

### Copy

AI is not a replacement for judgment. It is an instrument for repetitive work: reading messages, drafting replies, preparing estimates, organizing leads, updating records, and surfacing exceptions before they become expensive.

The businesses that win will not be the ones that buy the most tools. They will be the ones that install the clearest procedures.

## Visual notes for implementation

- Use a white cover with hard black type and red punctuation.
- Use dark interruption panels for doctrine and evidence.
- Use catalog/index grids for hubs and skills.
- Use square-edged cards and hard rules.
- Use mono metadata strips instead of soft badges.
- Use schematic diagrams instead of abstract AI imagery.
- Use AMTECH red only as signal, CTA, punctuation, and evidence accent.
