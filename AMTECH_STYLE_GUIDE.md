AMTECH AI — Internal CRM Production SpecificationVersion: 1.0
Stack: Next.js · Bolt Database · Bolt Auth · Resend · xAI Grok API
Deployment target: Bolt.new (vibe-code-first, agentic build)Product OverviewThis is an internal CRM for AMTECH AI, a B2B lead generation and outreach company. Two user types exist: one Admin (the founder) who manages leads, salespeople, lists, and offers; and Salespeople who work their assigned leads through a pipeline, take notes, and generate contracts.The Admin is not a database-level superuser — this distinction should inform UX, not backend permissions complexity. Keep auth simple.The most novel part of this data model is the list system: leads can belong to many lists, and lists can contain other lists. This is the core organizational primitive. Design it well — it's what makes the Admin's job fast.The most novel feature is AI contract generation: a salesperson picks a lead and an offer, and the app calls Grok 4.1 with structured reasoning output to produce a professional contract document. This should feel like a first-class feature, not a bolted-on afterthought.Design direction: Dark-first, utilitarian, fast. This is an internal tool used daily. Prioritize information density, keyboard accessibility, and instant feedback. Think sales ops software, not a marketing landing page.Phase 1 — Auth, Leads, and Admin FoundationGoal: A working app where the Admin can log in, upload leads from a CSV, and view them in a table. A salesperson can log in and see a placeholder workspace.What needs to exist when this phase is done:The database has a users table (with a role field distinguishing admin from salesperson) and a leads table (fields: business name, website, Google Maps listing, rating, contact name, phone, email). Bolt Auth handles login for both roles. Routes are protected by role. Magic link + email/password auth should both work via Bolt Auth — use whatever Bolt Auth supports natively without custom implementation.The Admin's main view is a leads table — searchable, sortable, with all fields visible. Above it, a CSV upload control: drag-and-drop or file picker, parses the expected columns, bulk-inserts rows, shows a result toast. Handle missing/malformed fields gracefully rather than failing the whole import.The Admin can also open an "Add Salesperson" panel, enter a name and email, create the user via Bolt Auth with the salesperson role, and trigger a Resend invite email with a magic link.The Salesperson landing page just needs to exist and be role-gated — an empty state is fine for Phase 1.Environment setup: At the end of Phase 1, output the complete list of required environment variables and what each one does.Phase 1 is done when: Admin uploads a CSV, sees rows populate the leads table, adds a salesperson, and that salesperson can log in.
