## Skill Design Takeaway

A future AMTECH article-writing skill should first create a draft artifact with validation metadata before touching live article code. The best pattern is:

```text
research docs + web sources
-> article validity check
-> graph role and entities
-> internal/external link plan
-> draft body
-> ArticleDefinition metadata
-> only then publish to the live article system
```

This avoids prematurely adding weak articles to the public graph and gives humans a reviewable research surface before generated OKF, routes, and Supabase projections are updated.
